# ETF雷达 - 产品需求文档（PRD）

## 项目概述

ETF雷达是一个前后端Web应用，用于自动追踪指定ETF基金的规模（份额）变化，帮助投资者直观感知大资金（如国家队）的动态。

> 💡 核心价值：将公开的ETF份额数据自动采集、汇总并可视化，替代手动Excel记录，提供更直观的趋势分析。

## 需求背景

- 大型ETF（如沪深300 ETF）的规模变化可以反映大资金的进出动态
- 数据源全部为公开数据，但手动收集效率低
- 需要一个自动化程序采集数据，并以图表形式展示趋势

## 功能需求

### P0 - 核心功能

#### F1: ETF份额数据采集

| 项目 | 说明 |
|------|------|
| 数据源 | 上交所/深交所官方接口 + 腾讯财经 |
| 采集方式 | 打开页面时自动增量更新 |
| 采集内容 | 指定ETF的每日份额（规模） |
| 存储 | SQLite 按日期存储每只ETF的份额数据 |

#### F2: ETF管理

- 支持用户自定义追踪的ETF列表（通过基金代码添加/删除）
- 默认内置主要宽基ETF（沪深300、中证500、上证50、创业板）
- 预置1461只ETF历史份额，新增追踪即有历史数据

#### F3: 数据展示与可视化

- 表格视图：按日期展示各ETF份额，支持查看历史数据
- 图表视图：
  - 单只ETF份额趋势折线图
  - 同类ETF份额汇总趋势图（如所有沪深300 ETF份额加总）
  - 支持总市值/份额切换
  - 支持分组切换（沪深300/中证500/上证50/创业板）

### P1 - 增强功能

#### F4: 数据分析

- 份额变化量/变化率计算
- 区间统计（周/月/季度净流入流出）
- 异常变动提醒（如单日份额变化超过阈值）

#### F5: 分组管理

- 支持按指数分组（沪深300组、中证500组等）
- 分组汇总统计

### P2 - 远期功能

#### F6: 消息推送

- 异常变动即时通知

#### F7: 数据导出

- 支持导出为 Excel / CSV

#### F8: 微信小程序

- API 已预留版本化接口，部署到公网后可接入

## 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 页面加载 < 2s，图表渲染流畅 |
| 可用性 | 增量更新失败不影响历史数据展示 |
| 数据准确性 | 上交所/深交所官方数据，每日精确 |
| 部署 | Docker 一键部署，零配置 |

## 数据源（已验证 2026-04-07）

| 数据源 | 用途 | 状态 | 备注 |
|--------|------|------|------|
| 上交所 sse.com.cn | 每日精确份额 | ✅ 可用 | 支持历史查询，不限流 |
| 深交所 szse.cn | 每日精确份额 | ✅ 可用 | 支持日期范围查询 |
| 腾讯财经 qt.gtimg.cn | 实时价格/市值 | ✅ 可用 | 批量请求，无限流 |
| 天天基金 FundSize API | - | ❌ 不可用 | 对场内ETF返回404 |

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据源不稳定 | 采集失败 | 多数据源备份 + 重试机制 |
| 数据源接口变更 | 采集逻辑失效 | 抽象采集层，便于快速适配 |
| 非交易日误采集 | 数据重复/错误 | 交易所接口非交易日自动返回空 |
